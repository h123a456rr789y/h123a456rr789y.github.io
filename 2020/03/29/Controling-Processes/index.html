Controling Processes

Posted on 2020-03-29 Edited on 2022-08-11
Introduction to Controling Process

Controling Process

  • Program to Process

    • program exacute -> process

  • Components of Process

    • Address space in mem
    • A set of DS within the kernel

  • Attributes of the Process

    • PID, PPID
    • UID, EUID
    • GID, EGID
    • Niceness

  • Process Lifecycle

    • fork: same PID
    • exac: exacute another programm with diff PID
    • exit: child use _exit to tell kernel that it is ready to end and child’s parent will know
    • wait: parent use wait to wait for child’s death

  • Attributes of the process – UID、GID、EUID and EGID

    • The effective uid and gid can be used to enable or restrict the additional permissions
    • Effective uid will be set to:
      • Real uid if setuid bit is off
      • The file owner’s uid if setuid bit is on

  • Signal

    • telling a process something has happened
    • Signals can be sent
      • communication between processes
      • kill, interrupt, or suspend process
        • 、
        • bg, fg
      • administrator to achieve various results
        • kill pid
        • killall
      • process violate the rules
        • divide by zero
        • Illegal memory access

  • Signal – Actions when receiving signal

    • Depend on whether there is a designated handler routine for that signal
    • “Catching” the signal
    • Two ways to prevent signals from arriving
      • Ignored
      • Blocked

  • FreeBSD signals

  • Signal – Send signals: kill

    • %kill [-signal] pid
    • ex:
      • kill -9 pid
      • kill -l
      • killall tcsh
      • killall -u harry

  • Niceness (Nice Value)

    • form -20 ~ +20 (-20 high to +20 low priority) related to CPU time quantum
    • Inherent Property
      • child niceless cant be higher than parent
    • Root has complete freedom in setting nice value
      • other users can only set for lower priority

  • Niceness – nice and renice

    • nice format
      • OS nice : $ /usr/bin/nice [range] utility [argument]
      • csh nice(built-in) : $ nice [range] utility [argument]
        • ex:$ nice +10 ps -l
      • renice format

  • cpuset command

    • To see how many CPUs on your machine
      • cpuset -g
    • Run commands with less CPUs
      • cpuset -l cpus cmd

  • Process States

  • ps command

    • ps
    • ps aux
    • ps auxww
    • a:all
    • w:wide

  • top command

    • top -q run top and renice it to -20
    • top -u don’t map uid to username
    • top -U show process owned by user
    • htop (advanced top)

  • Runaway process

    • Processes that use up excessive system resource or just go berserk
      • kill -TERM for unknown process
      • renice it to a higher nice value for reasonable process
